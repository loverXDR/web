Паттерн "Итератор" (Iterator Pattern)

Итератор — это поведенческий паттерн проектирования, который даёт возможность последовательно обходить элементы составных объектов, не раскрывая их внутреннего представления.

В PHP итераторы реализуются через интерфейсы Iterator и IteratorAggregate из стандартной библиотеки SPL (Standard PHP Library).

Диаграмма классов

Задание 1: Реализация интерфейса Iterator

В данном задании создан класс NumbersSquared, который реализует интерфейс Iterator для генерации последовательности квадратов чисел в заданном диапазоне.

Основные методы интерфейса Iterator:

  • rewind() - перемещает указатель в начало коллекции
  • valid() - проверяет, есть ли текущий элемент
  • current() - возвращает текущий элемент
  • key() - возвращает ключ текущего элемента
  • next() - перемещает указатель на следующий элемент

Задание 2: Реализация интерфейса IteratorAggregate

Во втором задании класс NewsDB был модифицирован для реализации интерфейса IteratorAggregate, что позволяет легко итерировать категории новостей.

Преимущества IteratorAggregate:

  • Упрощенная реализация - требуется только метод getIterator()
  • Возможность использования стандартных итераторов, таких как ArrayIterator
  • Удобный способ сделать класс итерируемым, не реализуя все методы интерфейса Iterator
