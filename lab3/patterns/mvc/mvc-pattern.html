Model-View-Controller (MVC) Pattern

The Model-View-Controller (MVC) is an architectural pattern that separates an application into three main logical components: the model, the view, and the controller. Each of these components are built to handle specific development aspects of an application.

Pattern Components

Model

The Model component corresponds to all the data-related logic that the user works with. It represents the data that is being transferred between the Controller and View components.

In our implementation:

  • Model: Base class for all models
  • UserModel: Specific model for user data

View

The View component is used for all the UI logic of the application. It generates output based on the model's data.

In our implementation:

  • View: Abstract base class for all views
  • HtmlView: Displays data as HTML
  • JsonView: Displays data as JSON
  • TextView: Displays data as plain text
  • MarkdownView: Displays data in Markdown format

Controller

The Controller acts on both Model and View. It controls the data flow into model objects and updates the view whenever data changes.

In our implementation:

  • Controller: Base controller class
  • UserController: Controller for user data with methods for different views
